## 04 - 1. 버블 정렬  
- ### 정의  
    - 두 인접한 데이터의 크기를 비교해 정렬하는 방법  
  
- ### 특징  
    - 구현이 간단하다.   
    - 시간 복잡도가 O(n^2)이라 다른 정렬 알고리즘보다 속도가 느리다.  
  
## 04 - 2. 선택 정렬  
- ### 정의  
    - 최대나 최소 데이터를 데이터가 나열된 순으로 찾아가며 선택하는 방법  
  
- ### 특징  
    - 구현이 복잡하다.  
    - 시간 복잡도가 O(n^2)이라 다른 정렬 알고리즘보다 속도가 느리다.  
  
## 04 - 3. 삽입 정렬  
- ### 정의  
    - 이미 정렬된 데이터 범위에 정렬되지 않은 데이터를 적절한 위치에 삽입하는 방법  
  
- ### 특징  
    - 구현이 간단하다.  
    - 시간 복잡도가 O(n^2)이라 다른 정렬 알고리즘보다 속도가 느리다.
